Network Operating Systems - Summary

Overview

This document covers lectures on Operating Systems in general and Network Operating Systems in particular. It provides comprehensive information about OS services, process management, memory management, storage, protection, security, and different computing environments.

1. Operating System Services

Operating systems provide an environment for execution of programs and services to programs and users:

  • User Interface - CLI, GUI, touch-screen, Batch
  • Program Execution - Loading programs into memory and running them
  • I/O Operations - Management of input/output devices
  • File-System Manipulation - Reading, writing, creating, deleting files
  • Communications - Inter-process communication
  • Error Detection - Hardware, software, and user program errors
  • Resource Allocation - CPU cycles, memory, file storage, I/O devices
  • Protection and Security - Controlling access to resources

2. Computer System Structure

Computer system can be divided into four components:

  • Hardware - Provides basic computing resources (CPU, memory, I/O devices)
  • Operating System - Controls and coordinates use of hardware
  • Application Programs - Define how system resources are used
  • Users - People, machines, other computers

3. Process Management

A process is a program in execution. It is a unit of work within the system.

  • Program is a passive entity, process is an active entity
  • Process needs resources (CPU, memory, I/O, files)
  • OS activities: creating/deleting processes, suspending/resuming, synchronization, communication, deadlock handling

4. Memory Management

Memory management determines what is in memory and when:

  • Optimizing CPU utilization and computer response
  • Keeping track of memory usage
  • Deciding which processes to move in/out of memory
  • Allocating and deallocating memory space

5. Storage Management

Storage systems organized in hierarchy based on speed, cost, and volatility:

  • Main Memory - Random access, volatile, DRAM
  • Secondary Storage - Nonvolatile, large capacity (HDD)
  • Non-volatile Memory (NVM) - Faster than HDD, nonvolatile
  • Caching - Copying information into faster storage

6. Protection and Security

  • Protection - Mechanism for controlling access to resources
  • Security - Defense against internal and external attacks
  • User identities (user IDs, security IDs)
  • Group identifier (group ID)
  • Privilege escalation

7. Computing Environments

  • Traditional - Stand-alone machines, network computers, mobile computers
  • Mobile - Smartphones, tablets with extra features (GPS, gyroscope)
  • Client-Server - Servers responding to client requests
  • Cloud Computing - Computing, storage, apps as a service

8. System Calls

Programming interface to the services provided by the OS:

  • Typically written in high-level language (C or C++)
  • Accessed via APIs (Win32 API, POSIX API, Java API)
  • Types: Process control, File management, Device management, Information maintenance, Communications, Protection
